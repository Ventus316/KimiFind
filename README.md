## 🎯 2. 專案動機與核心理念

- **⚡ 痛點解決**：桃園市民、大學生及旅遊者在日常移動中遺失物品時，傳統招領管道過於破碎（散落在各類 Facebook 社群、Dcard 校園板或官方公告）。

- **🌟 集中化平台**：KimiFind 建立了一個**結構化、區域限定**的失物歸還系統，涵蓋桃園市各大行政區，落實互助精神。

- **🌌 浪漫視覺風格**：

- **逢魔時刻 (Twilight Gradient)**：全站背景採用紫紅色過渡到深藍色的黃昏星空漸層，呼應跨越時空尋覓的氛圍。

- **結繩紅 (Musubi Red)**：主按鈕與強調樣式採用亮紅色（`#E63946`），象徵牽引失主與拾獲者緣分的紅線。

- **毛玻璃質感 (Glassmorphism)**：大量運用 Tailwind CSS 的 `backdrop-blur`，讓資訊卡片顯得清新透明。

## ⚡ 3. 網頁互動五大核心技術特性

本網頁為**純前端架構（無 Node.js 或伺服器資料庫）**，透過 **jQuery 操作 DOM** 與 **LocalStorage 狀態管理**，實現了媲美全端 App 的流暢體驗：

1. **🔐 前端狀態管理與路由防護 (State Management & Route Guards)** \* 利用 `localStorage` 模擬完整的登入/註冊系統。系統會根據當前使用者的 `role` (一般用戶 vs. 管理員) 動態渲染 Navbar，若未登入或越權訪問後台網址，將會被路由守衛攔截並強制跳轉。
2. **🔄 跨頁面資料連動 (Cross-Page Data Sync)** \* 完美實作無後端的 CRUD 邏輯。使用者上報的新遺失物、或管理員新增的「分類標籤」，皆會即時同步至首頁、搜尋頁與各表單下拉選單中。
3. **🔍 多維度動態篩選 (Multi-dimensional Dynamic Filtering)** \* 在搜尋頁面中，支援透過「關鍵字」、「物品狀態(遺失/拾獲)」、「行政區與類別」進行多條件交叉比對，並利用 jQuery `.fadeOut()` 與 `.fadeIn()` 進行無縫的卡片重新渲染。
4. **💬 動態節點生成與互動 (Dynamic Node Generation)** \* 物品詳情頁支援動態新增留言、以及透過 LocalStorage 紀錄個別使用者的「點讚」與「收藏」狀態，並能連動至個人後台的收藏庫。
5. **🛡️ 管理員黑名單機制 (Admin Takedown System)** \* 管理員後台可對違規貼文執行「強制下架」，下架物品之 ID 將寫入黑名單，全站（首頁、搜尋頁）將透過進階的陣列過濾 `.filter()` 瞬間將其屏蔽。

## 🗂️ 4. 專案檔案目錄結構

專案採用扁平、模組化的純前端目錄結構，便於多人協作與獨立維護：

```text
KimiFind/
├── index.html              # 網站首頁：品牌傳達、最新遺失物動態輪播
├── search.html             # 失物搜尋：行政區與類別條件動態篩選
├── detail.html             # 物品詳情：網址參數解析、點讚收藏與留言
├── report.html             # 失物上報：引導式表單填寫、前端防呆驗證
├── dashboard.html          # 個人後台：動態 Modal 編輯、狀態切換、收藏管理
├── guide.html              # 歸還認領：垂直時間軸流程指南
├── login.html              # 會員登入：測試帳號與權限模擬
├── register.html           # 會員註冊：前端註冊狀態模擬
├── admin.html              # 管理員後台：違規下架、用戶升降級、類別管理
├── navbar.html             # 共用元件：動態權限導覽列 (由 jQuery 動態載入)
├── css/
│   └── style.css           # 客製化樣式：字體與微調
├── js/
│   ├── data.js             # 模擬資料庫：預設的假資料與初始狀態設定
│   ├── main.js             # 首頁邏輯：資料合併、黑名單過濾與渲染
│   ├── search.js           # 搜尋邏輯：動態標籤生成、多維度交叉過濾
│   ├── detail.js           # 詳情邏輯：URL 參數擷取、互動狀態儲存
│   ├── report.js           # 上報邏輯：資料庫寫入、動態類別選單讀取
│   ├── dashboard.js        # 個人後台：CRUD 實作與事件代理 (Event Delegation)
│   ├── admin.js            # 管理後台：三大管理模塊與黑名單寫入
│   └── nav.js              # 導覽列邏輯：元件動態載入、登出機制與高亮當前頁
└── images/
    └── items/              # 測試資料高畫質實體圖片資源 (含 default.jpg)
```

## 💾 5. 模擬資料庫結構設計 (Mock DB)

在 `js/data.js` 中定義了 5 筆具備豐富故事背景、緊扣主題並融入桃園在地地標的假資料。資料結構具備完整的擴充性：

- 包含 `type` (遺失/拾獲)、`status` (尋找中/已結案) 作為核心分類。
- 包含 `publisherEmail` 綁定特定使用者，以及 `contact` (Line ID / 手機) 以供詳情頁展示。

## 🚀 6. 如何在本機運行與測試

由於本專案為**純前端應用程式**，不需安裝任何後端環境（如 Node.js 或資料庫），即可直接在任何瀏覽器中完美執行。

### 方案 A：線上直接體驗 (推薦)

請直接點擊專案的 GitHub Pages 連結：**[KimiFind 線上展示預覽](https://ventus316.github.io/KimiFind/)**

### 方案 B：本機端運行 (Local Testing)

⚠️ **注意**：由於本專案使用了 jQuery 的 `.load()` 動態匯入共用導覽列 (`navbar.html`)，若直接雙擊 HTML 檔案 (`file:///`)，會因瀏覽器的 **CORS 安全性限制** 導致導覽列無法顯示。

1.  請下載整包專案程式碼。
2.  使用 **VS Code** 開啟專案資料夾。
3.  安裝並點擊 **Live Server** 擴充套件來運行 `index.html`。

### 🎬 黃金測試劇本 (Demo Script)

1.  **身分體驗**：前往「登入頁」，使用 `user@kimi.com` (密碼 `123456`) 登入為一般用戶，或使用 `admin@kimi.com` (密碼 `123456`) 登入為管理員。
2.  **上報與首頁連動**：前往「失物上報」新增一筆遺失物，送出後返回首頁，會發現剛上報的物品已動態出現在「剛落下的星辰」區塊。
3.  **個人後台 CRUD**：點擊右上角名字進入「個人後台」，嘗試利用彈出視窗(Modal)「✏️修改名稱」、切換尋獲狀態，並前往搜尋頁隨意「收藏」物品後回後台查看。
4.  **管理員制裁 (需使用 admin 登入)**：前往「管理員後台」，點擊某一筆物品的「強制下架」。隨後回到搜尋頁與首頁，會發現該物品已在全站徹底消失！
