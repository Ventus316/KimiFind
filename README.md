## 🎯 2. 專案動機與核心理念

- **⚡ 痛點解決**：桃園市民、大學生及旅遊者在日常移動中遺失物品時，傳統招領管道過於破碎（散落在各類 Facebook 社群、Dcard 校園板或官方公告）。
- **🌟 集中化平台**：KimiFind 建立了一個**結構化、區域限定**的失物歸還系統，涵蓋桃園市各大行政區，落實互助精神。
- **🌌 浪漫視覺風格**：
  - **逢魔時刻 (Twilight Gradient)**：全站背景採用紫紅色過渡到深藍色的黃昏星空漸層，呼應跨越時空尋覓的氛圍。
  - **結繩紅 (Musubi Red)**：主按鈕與強調樣式採用亮紅色（`#E63946`），象徵牽引失主與拾獲者緣分的紅線。
  - **毛玻璃質感 (Glassmorphism)**：大量運用 Tailwind CSS 的 `backdrop-blur`，讓資訊卡片顯得清新透明。

## ⚡ 3. 網頁互動三大核心技術特性

本網頁為**純前端架構（無後端伺服器）**，所有動態資料操作皆由 **jQuery 操作 DOM 節點** 與記憶體資料庫流暢完成：

1.  **🔍 動態內容替換 (Dynamic Content Replacement)**
    - _應用場景_：失物搜尋頁 (`search.html`)
    - _技術實現_：當使用者輸入關鍵字或點擊地區/類別篩選標籤時，jQuery 會攔截事件並觸發 `.fadeOut()` 隱藏舊清單，隨即執行 `.empty()` 清空節點，透過 JavaScript 篩選陣列後重新組裝 HTML，最後以 `.fadeIn()` 動態渲染符合條件的全新物品卡片。
2.  **💬 動態產生節點 (Dynamic Node Generation)**
    - _應用場景_：物品詳情頁 (`detail.html`)
    - _技術實現_：使用者在留言區輸入文字並發送時，系統會即時擷取內容並組合為帶有 Tailwind 樣式的 HTML 區塊，利用 `.prepend()` 無縫安插至留言列表最上方，並搭配 `.slideDown(300)` 動態展現，不需重新整理網頁。
3.  **🗑️ 動態刪除節點 (Dynamic Node Removal)**
    - _應用場景_：個人管理後台 (`dashboard.html`)
    - _技術實現_：拾獲者管理發布紀錄時，點擊垃圾桶會觸發內建安全確認。確認後，jQuery 透過 `.closest()` 定位最外層卡片節點，執行 `.fadeOut(400)` 漸隱動畫，隨後呼叫 `.remove()` 將其從網頁 DOM 樹中徹底移除。

## 🗂️ 4. 專案檔案目錄結構

專案採用扁平、模組化的純前端目錄結構，便於多人協作與獨立維護：

```text
KimiFind/
├── index.html              # 網站首頁：品牌傳達、最新遺失物動態輪播
├── search.html             # 失物搜尋：行政區與類別條件動態篩選
├── detail.html             # 物品詳情：網址參數解析、動態產生留言節點
├── report.html             # 失物上報：引導式表單填寫、前端防呆驗證
├── dashboard.html          # 個人管理後台：狀態更新、動態刪除卡片
├── guide.html              # 如何歸還＋認領：垂直時間軸流程指南
├── auth.html               # 帳號認證：沉浸式毛玻璃註冊與登入頁
├── admin.html              # 管理員頁面：違規貼文強制下架數據表格
├── about.html              # 關於我們：資傳系團隊理念介紹與成員卡片
├── css/
│   └── style.css           # 客製化樣式：存放特殊星空動畫與粒子特效
├── js/
│   ├── data.js             # 模擬資料庫：全站唯一、共用的 5 筆預設 JSON 陣列
│   ├── main.js             # 共用與首頁邏輯：Navbar 互動與最新 3 筆物品渲染
│   ├── search.js           # 搜尋頁邏輯：關鍵字與 Pills 標籤雙重條件篩選
│   ├── detail.js           # 詳情頁邏輯：URLQuery 解析與動態留言安插
│   ├── report.js           # 上報頁邏輯：文字欄位空值捕獲與紅色警告回饋
│   └── dashboard.js        # 後台邏輯：事件代理機制與 DOM 卡片淡出刪除
└── images/
    └── items/              # 測試資料高畫質實體圖片資源 (musubi, book, sketch 等)
```

## 🚀 6. 如何在本機運行與測試

由於本專案為**純前端應用程式**，不需安裝任何後端環境（如 Node.js 或資料庫），即可直接在任何瀏覽器中完美執行。

### 📦 步驟 A：下載與配置

1.  將本儲存庫 `clone` 或下載解壓縮至您的電腦本機。
2.  確保圖片資源已正確放置於 `images/items/` 目錄下，且檔名與 `data.js` 保持一致。

### 🎬 步驟 B：正確的測試點擊劇本 (Demo Script)

為了體驗最完整的動態資料傳遞：

1.  **不要直接點開 `detail.html`**（否則會因為網址無參數而無法撈取資料）。
2.  請直接雙擊打開 **`index.html`** 進入首頁。
3.  點擊頂部導覽列前往「失物搜尋」，嘗試點擊標籤切換，體驗**動態內容替換**。
4.  點擊任何卡片下方的「查看詳情」，此時網址列會自動帶入 `?id=item_00X`，即可在詳情頁看到對應的物品圖片與資料！
5.  在詳情頁最下方留言板輸入文字並點擊發送，體驗**動態產生節點**。
